## Codebase Patterns (carried over from Wave 1, append new findings here)
- Track ScrollView scroll position with a named `.coordinateSpace(name:)` on the ScrollView + a 1pt `GeometryReader`-backed anchor view at the end of its content that emits its `.frame(in: .named(...)).minY` via a `PreferenceKey`. Parent reads both the anchor minY and the ScrollView's visible height (from the outer GeometryReader's `geo.size.height`) to derive "near bottom" ÔÇö no ScrollViewReader needed for that. Works on iOS 16.
- When removing a card from the timeline that was used as the pre-compile entry (e.g. CompilePromptCard), leave the DailyPageEntryCard post-compile branch in place and move the pre-compile entry to a sticky footer. Do NOT keep the card as a dual-state widget ÔÇö parent conditionals stay cleaner.
- Mount sticky chrome (compile footer, etc.) as a sibling of the ScrollView inside the outer VStack, ABOVE the InputBarView, so the keyboard and input bar layout are untouched.
- Build command: `xcodebuild -scheme DayPage -destination 'platform=iOS Simulator,arch=arm64,id=3A661BD8-5F57-42B2-B4BA-DBB2FAB4FAD0' build`
- SourceKit diagnostics show macOS errors for AVAudioSession ÔÇö these are iOS-only APIs and build fine in the iOS target
- SourceKit also misreports "Cannot find X in scope" for cross-module symbols ÔÇö build succeeds regardless
- VoiceService is @MainActor singleton; reuse VoiceService.transcribe(audioURL:) for the new transcribe-on-swipe-left path (US-008)
- DSColor / DSFonts / DSRadius / DSSpacing tokens used for all visuals
- VaultInitializer.vaultURL is the entry point for vault file paths
- To add new Swift files: add PBXBuildFile + PBXFileReference + group child + Sources entry in project.pbxproj (use sequential IDs)
- DayPageLogger uses nonisolated static helpers called from a DispatchQueue to avoid @MainActor actor isolation issues
- DashScope OpenAI-compatible endpoint is `https://dashscope.aliyuncs.com/compatible-mode/v1` (NOT `coding.dashscope.aliyuncs.com/v1` ÔÇö that returns 404). The `Secrets.dashScopeBaseURL` env override still wins; only the in-code default changed.
- Use `#if DEBUG print(...)` (not DayPageLogger) for noisy network diagnostics ÔÇö keeps release logs clean and avoids @MainActor isolation hops.
- For archive-style detail screens that read vault files: load asynchronously inside a `.task` closure into an explicit `LoadState` enum, not inside computed properties. Computed-property IO runs on every re-render and hides failure paths.
- Validate user-supplied date strings with BOTH a strict `^\d{4}-\d{2}-\d{2}$` regex AND a non-lenient DateFormatter round-trip ÔÇö the regex alone lets `2020-02-30` through.
- When a file-backed view (RawMemoView, DailyPageView) is opened, double-check `FileManager.default.fileExists(atPath:)` inside its own load path even if the parent already checked. Log `url.path` + `errno` on any miss ÔÇö both are necessary to diagnose sandbox / migration issues.
- When adding a "test this endpoint" Settings action, ALWAYS resolve baseURL/model the same way the production service does (`Secrets.*.isEmpty ? defaultLiteral : Secrets.*`). If you hardcode a URL in Settings, you will silently drift from CompilationService and the test can green while real compile 404s.
- NSURLError codes (timeout=-1001, notConnected=-1009, cannotFindHost=-1003, cannotConnect=-1004) are the right bucket for network-layer hints; HTTP status codes only show up when the server actually responded.
- "Single source of truth for in-flight UI state": when collapsing redundant indicators, also delete the timed `Task` chains that only existed to refresh the now-removed banner. They become dead code (`kind == .progress` guards never fire), and leaving them in the file plants traps for future readers. Either fully delete or pipe their signal into the surviving indicator's view-model field.
- BannerCenter is the right channel for **terminal** outcomes (success / offline / error) but a poor fit for in-flight progress ÔÇö it competes with timeline cards. Reserve it for "the operation is done, here is the verdict" messages.
- For off-main vault scans, put the real work in a `fileprivate enum`'s `static func` at file scope and call it from inside `Task.detached { ... }.value`. Instance methods fail to compile (`self` capture) and `Self.foo` also fails in detached contexts ÔÇö file-scope statics are the only clean escape hatch.
- Calendar cell "has data" state should be derived from a pre-scanned `Set<String>` of date basenames (cheap `.contains`), NOT from per-day `DayStats` aggregations (which read file *contents*). Split the two axes: "is anything here?" (pre-scan) vs. "how much activity?" (heatmap aggregation).
- When a detail view already has an explicit `LoadState` handling `.empty`/`.error`/etc, parent cell/list taps should push unconditionally and delete any pre-tap `.alert` guards. Alerts that were protecting a crash-prone detail view become dead UX once the detail view is hardened.
- Feature-flag toggles whose PRD copy is phrased against the legacy path (e.g. 'ńŻ┐šöĘŠŚžšëłŔżôňůąŠíć') should keep the storage key positive (`useInputBarV2 = true` by default) and invert only inside the Settings Toggle's Binding (`get: { !key }, set: { key = !$0 }`). Avoids 'off means on' confusion later.
- For local popover/menu tap-outside dismissal bounded to a parent view (InputBar, card), a `Color.black.opacity(0.001).contentShape(Rectangle()).onTapGesture { close() }` layer gated by the open-state Bool is enough ÔÇö no need for a full-screen overlay or DragGesture. Put it in `.background` so it sits behind the popover but in front of the parent view's main content.
- `.photosPicker(isPresented:selection:matching:photoLibrary:)` is the iOS-16 "imperative" API ÔÇö trigger the picker from any callback closure without having to embed a `PhotosPicker` label view. Pair with `onChange(of: photosPickerItem)` to forward the selection.
- For press-to-talk gestures, use `DragGesture(minimumDistance: 0)` + `@GestureState` for auto-reset, BUT capture the latest translation into a separate `@State CGSize` inside `.onChanged` ÔÇö some iOS versions reset the gesture's translation before `.onEnded` fires, so reading from `value.translation` in `.onEnded` can give `.zero`. Fall back to the last captured translation when `value.translation == .zero`.
- Fire first-edge haptics (press-down) inside `.onChanged`'s first invocation by gating on `currentPhase == .idle` then transitioning to `.recording`. `.onChanged` has no "did begin" hook ÔÇö the first call is the begin.
- Attach press-to-talk gestures via `.highPriorityGesture(...)` (not `.gesture` / `.simultaneousGesture`) so an outer ScrollView's DragGesture doesn't steal the touch. `.simultaneousGesture` would let BOTH fire, causing the ScrollView to scroll while the user is holding the button.
- When two gesture thresholds are crossed simultaneously (e.g. diagonal up-left), pick the "safer" outcome as the tiebreaker. For press-to-talk: cancel wins over transcribe when `|dy| >= |dx|` ÔÇö users expect "discard" on ambiguous gestures, not "send the transcript".
- For a floating in-input-bar overlay that tracks live audio, mount it as the FIRST child of the InputBar's VStack, gated by gesture-phase state. This keeps the overlay above the text field but anchored to the input bar so `ignoresSafeArea` and keyboard rules don't need any changes.
- The transcribe-only path must `try? FileManager.default.removeItem(at: result.fileURL)` + `voiceService.reset()` after reading `result.transcript` ÔÇö otherwise the audio file lingers in `vault/raw/assets/` and the Whisper queue might retry it later. Only the send path should keep the audio file.

- **Learnings for future iterations:**
  - `DragGesture(minimumDistance: 0)` is the only clean way to get both press-down and release in one gesture recognizer. Tap + LongPress + DragGesture combined via `.simultaneously(with:)` creates race conditions ÔÇö the minimumDistance-0 drag handles everything.
  - `@GestureState` resets automatically when the gesture ends, but reading `value.translation` in `.onEnded` can return `.zero` on some iOS builds. Always keep a separate `@State lastTranslation: CGSize` that you update in every `.onChanged` and fall back to it on release.
  - `.highPriorityGesture` is the correct attachment for press-to-talk inside a scrollable parent. `.gesture` and `.simultaneousGesture` let the outer ScrollView's DragGesture fire too, causing scroll-during-hold bugs.
  - Haptic feedback is best done with `UIImpactFeedbackGenerator(style:)` created per-edge with `.prepare()` called just before `.impactOccurred()`. Don't try to reuse a single generator across phases ÔÇö the warm-up window is part of the API contract, and reusing can drop haptics on fast transitions.
  - When a gesture has multiple armed states (cancel / transcribe), pick the "safer" outcome as the tiebreaker for ambiguous diagonals. Here: cancel wins when `|dy| >= |dx|`. Users get upset when a slightly-up-and-left swipe sends a transcript instead of cancelling.
  - For an overlay that needs live data from a `@StateObject` service, prefer putting the `@StateObject` on the view that mounts the overlay (InputBarV2) rather than passing the service down through the callback shape. The parent-view callbacks stay pure (result in / transcript in) and the overlay reads directly from the service via SwiftUI's binding.
  - The transcribe-only path must explicitly delete the audio file (`FileManager.default.removeItem(at:)`) because `VoiceService.stopAndTranscribe()` writes it to `vault/raw/assets/` for the send path to consume. If you skip the delete, the next `VoiceAttachmentQueue` pass may retry the orphaned audio file.
