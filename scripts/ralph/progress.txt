## Codebase Patterns
- `project.pbxproj` 手动管理：新增 Swift 文件必须同步添加 PBXFileReference（A2xxxxxxx）、PBXBuildFile（A1xxxxxxx），并加入目标 Group children 和 Sources build phase files
- `xcrun swiftc -typecheck <files> -sdk $(xcrun --sdk iphonesimulator --show-sdk-path) -target arm64-apple-ios16.0-simulator` 是本项目的 typecheck 命令
- Vault 根目录通过 `VaultInitializer.vaultURL` 获取（Documents/vault/），所有 Storage 层代码应使用此统一入口
- 首次初始化写入：SCHEMA.md、wiki/index.md、wiki/hot.md、wiki/log.md 四个种子文件
- API 密钥通过 `Secrets.xxx` 静态常量访问（dashScopeApiKey / dashScopeBaseURL / dashScopeModel / openAIWhisperApiKey / openWeatherApiKey）；GeneratedSecrets.swift 由构建脚本在每次编译时从 .env 重新生成
- `.gitignore` 中忽略路径需写完整相对路径：`DayPage/Config/GeneratedSecrets.swift`（不能只写 `Config/GeneratedSecrets.swift`，那样只能匹配根目录下的 Config）
- PBXShellScriptBuildPhase 添加方式：在 PBXShellScriptBuildPhase section 中定义，然后在 PBXNativeTarget 的 buildPhases 数组中（放在 Sources 之前）引用
